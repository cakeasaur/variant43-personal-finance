## 1 Аналитическая часть — анализ аналогов

Ниже приведён обзор популярных решений для ведения личных финансов. Цель —
выделить ключевые функции, подходы к UX и практики безопасности, чтобы сформировать
обоснованные требования к нашему приложению.

### 2.1 Список аналогов (для сравнения)
Для анализа выбраны приложения с широким набором функций:
- **Zenmoney** (учёт, категории, бюджеты, цели; часто с синхронизацией)
- **CoinKeeper** (упор на UX, визуализация, быстрый ввод)
- **Monefy** (простота, быстрый учёт, базовые отчёты)
- **Spendee** (категории, визуализация, бюджеты/кошельки)
- **Money Manager** (широкие отчёты, учёт долгов/счетов)

### 2.2 Сравнение по функционалу (качественно)
- **Учёт доходов/расходов**: присутствует во всех аналогах; различия — скорость ввода и гибкость фильтров.
- **Категоризация**: везде есть, часто — иерархия категорий и цветовые метки.
- **Визуализация**: обычно круговые диаграммы по категориям + график по времени.
- **Цели**: поддерживаются не во всех; чаще встречаются в более «полных» бюджетных приложениях.
- **Напоминания**: обычно как «планируемые платежи»/повторяющиеся операции.
- **Безопасность**: PIN/биометрия встречаются часто; шифрование данных — редко описано прозрачно, но ожидается защита локального хранилища.

### 2.3 UX-наблюдения (что полезно перенять)
- **Быстрый ввод**: минимальное число действий для добавления расхода (сумма → категория → сохранить).
- **Последние категории**: ускоряют категоризацию «повторяющихся» покупок.
- **Период по умолчанию**: чаще всего текущий месяц.
- **Отчёты**: переключение периода (месяц/неделя/год) и drill-down до списка транзакций.

### 2.4 Выводы и решения для нашего проекта
В рамках варианта 43 (Python + Kivy + SQLite) и требований методички целесообразно:
- сделать **офлайн-first** с локальной SQLite БД;
- реализовать **круговую диаграмму** расходов по категориям и **линейный график** динамики;
- включить **цели** и **напоминания** как отдельные сущности;
- обеспечить **защиту данных** через шифрование (как обязательную часть варианта);
- сфокусироваться на **быстром добавлении транзакций** и простых фильтрах.
