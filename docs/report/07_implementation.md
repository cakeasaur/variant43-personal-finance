## 4 Реализация

### 4.1 Структура проекта
Код расположен в каталоге `src/` и разделён на уровни:
- `src/app.py` — точка входа Kivy, UI (транзакции, отчёты, цели, напоминания), сценарии взаимодействия;
- `src/core/` — доменные модели и расчёты агрегатов (отчёты);
- `src/infra/db/` — подключение SQLite, схема и репозитории;
- `src/infra/security/` — криптография для защиты локальной БД.

### 4.2 Основные сценарии
- **Транзакции**: добавление дохода/расхода, фильтр по типу, отображение списка и итогов за месяц.
- **Отчёты**: построение диаграмм по категориям и по дням (агрегации доменного слоя + визуализация в UI).
- **Цели**: CRUD целей, прогресс (current/target), опциональный дедлайн.
- **Напоминания**: CRUD напоминаний, дата и периодичность (none/daily/weekly/monthly), отметка выполнения.

### 4.3 Хранение данных (SQLite)
Инициализация схемы выполняется в `src/infra/db/schema.py`, доступ к данным — через репозитории
в `src/infra/db/repositories.py`. Схема включает таблицы:
`categories`, `transactions`, `goals`, `reminders`, `settings`.

### 4.4 Защита данных
Реализовано шифрование файла БД:
- контейнер AES-GCM;
- ключ из пароля через scrypt;
- ввод пароля при запуске приложения;
- пере-шифрование при завершении работы приложения (best-effort).
