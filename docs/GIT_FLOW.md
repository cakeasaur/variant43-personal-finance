## Git flow (упрощённая схема для проекта)

### Ветки
- **main**: стабильная ветка, только релизы (теги)
- **develop**: интеграционная ветка для текущей разработки
- **feature/<кратко>**: новая функциональность
- **fix/<кратко>**: исправления дефектов
- **docs/<кратко>**: документация/записка/диаграммы

### Правила
- Разработка ведётся в `feature/*` / `fix/*`, затем merge в `develop`.
- Перед релизом: стабилизация в `develop`, затем merge в `main` и постановка тега.
- В `main` не коммитим напрямую.

### Именование веток (примеры)
- `feature/transactions-crud`
- `feature/charts`
- `fix/db-migration`
- `docs/report-structure`
